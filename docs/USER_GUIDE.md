## 1. 快速开始

### 1.1 系统要求

在安装和运行 Outlook Auto-Archiver 之前，请确认您的电脑满足以下条件：

| 项目 | 要求 |
|------|------|
| 🖥️ 操作系统 | Windows 10 / Windows 11 |
| 📧 邮件客户端 | Microsoft Office LTSC 2024 专业增强版 或 经典版 Classic Outlook |
| 💾 磁盘空间 | PST 存储目录建议预留至少 5GB 可用空间 |
| 🔐 运行权限 | 当前普通用户权限即可，**无需管理员权限** |

> ⚠️ **重要提示**：本程序**不支持**基于 WebView2 的"新版 Outlook"（New Outlook），仅支持传统 COM 接口的 Classic Outlook。请确保您使用的是经典版 Outlook。

### 1.2 获取程序

#### 方式一：下载预编译版本（推荐）

从 [Releases](https://github.com/deavorwei/outlookTool/releases) 页面下载最新的 `outlook-archiver.exe` 文件。

#### 方式二：从源码编译

如果您的电脑已安装 [Go 1.26+](https://go.dev/dl/)，可以自行编译：

```powershell
# 1. 克隆仓库
git clone https://github.com/deavorwei/outlookTool.git
cd outlookTool

# 2. 编译（优化体积）
go build -ldflags="-s -w" -o outlook-archiver.exe .
```

#### 部署建议

> 💡 **建议**：将 `outlook-archiver.exe` 放置在**空间充足**的磁盘分区（如 `D:\Tools\`），因为程序会在同目录下生成配置文件、日志目录和 PST 存储目录。

### 1.3 首次运行

1. **确保 Outlook 已启动**：程序需要通过 COM 接口与 Outlook 交互，因此运行前请先打开 Classic Outlook 并登录您的 Exchange 邮箱。

2. **双击运行**：直接双击 `outlook-archiver.exe`，或在 PowerShell 中执行：

   ```powershell
   .\outlook-archiver.exe
   ```

3. **观察系统托盘**：程序启动后会在任务栏右下角的系统托盘区域显示一个 📥 图标。

4. **自动生成文件**：首次运行时，程序会自动在 exe 同目录下生成以下内容：

   ```
   📁 程序目录/
   ├── outlook-archiver.exe    # 程序主体
   ├── config.yaml             # 配置文件（自动生成）
   ├── logs/                   # 日志目录（自动生成）
   │   └── archiver.log        # 运行日志
   └── OutlookArchives/        # PST 存储目录（自动生成）
   ```

> 💡 **提示**：如果程序无法启动，请先检查 Outlook 是否已启动并正常运行，详见 [5. 故障排除](#5-故障排除)。

### 1.4 建议的首次运行流程（Dry Run）

> ⚠️ **强烈建议**：首次使用时，请先以 **Dry Run（模拟运行）** 模式运行，确认归档计划符合预期后再正式执行。

**步骤如下：**

1. 打开程序目录下的 `config.yaml` 文件（可用记事本或任何文本编辑器）。

2. 找到 `dry_run` 配置项，将其设置为 `true`：

   ```yaml
   dry_run: true
   ```

3. 保存文件后，通过系统托盘右键菜单选择 🔄 **重新加载配置**（或重启程序）。

4. 等待程序执行一轮归档扫描（默认每 10 分钟一次），然后查看日志文件 `logs/archiver.log`。

5. 在日志中，您会看到类似以下的模拟输出：

   ```
   [INFO] [DryRun] Would move: "项目周报" (2024-03-15) → OutlookArchives/2024_Q1.pst
   [INFO] [DryRun] Would move: "客户反馈" (2024-06-22) → OutlookArchives/2024_Q2.pst
   ```

6. 确认日志中的归档目标和邮件列表正确无误后，将 `dry_run` 改回 `false` 并重新加载配置：

   ```yaml
   dry_run: false
   ```

### 1.5 验证归档结果

归档执行后，您可以通过以下方式验证结果：

1. **在 Outlook 中查看 PST 文件**：
   - 打开 Outlook → 点击 **文件** → **打开和导出** → **打开 Outlook 数据文件**
   - 浏览到 `OutlookArchives` 目录，选择对应的 PST 文件（如 `2024_Q2.pst`）
   - PST 文件会在 Outlook 侧边栏中显示，您可以像浏览普通文件夹一样查看归档的邮件

2. **检查日志文件**：打开 `logs/archiver.log`，查看归档操作的详细记录。

3. **确认 PST 文件已生成**：在 `OutlookArchives` 目录下，应该能看到按季度命名的 PST 文件，例如：

   ```
   📁 OutlookArchives/
   ├── 2023_Q4.pst
   ├── 2024_Q1.pst
   ├── 2024_Q2.pst
   └── 2024_Q3.pst
   ```

## 2. 功能详解

### 2.1 自动定时归档

自动定时归档是本程序的核心功能。程序会在后台静默运行，按设定的间隔自动扫描邮箱，将超过保留天数的历史邮件按季度归档到本地 PST 文件。

#### 工作原理

1. ⏰ **定时触发**：程序按 `poll_interval_minutes`（默认 10 分钟）的间隔自动启动归档扫描。
2. 📂 **扫描文件夹**：遍历邮箱中的所有文件夹（自动跳过"已删除邮件"、"草稿"等系统文件夹）。
3. 🔍 **过滤邮件**：对每个文件夹，筛选出超过 `retain_days`（默认 30 天）的邮件。
4. 📊 **分块处理**：按时间降序分块处理邮件（块大小由 `stream_block_size` 控制），避免一次性加载过多数据。
5. 📅 **季度路由**：根据邮件的**实际发送/接收时间**计算所属季度（如 2024 年第二季度），生成对应的 PST 文件名（如 `2024_Q2.pst`）。
6. 📁 **确保目录**：自动在目标 PST 中创建与源邮箱一致的文件夹层级结构。
7. ➡️ **执行移动**：将邮件从 Exchange 邮箱移动到对应的 PST 文件中。
8. 🛡️ **安全延迟**：执行前会等待 `safe_delay_minutes`（默认 10 分钟），防止与 Exchange 服务器同步产生冲突。

#### 归档规则

**季度划分方式**：

程序严格按照邮件的真实发送/接收时间（而非系统当前时间）来决定归档到哪个季度 PST 文件：

| 季度 | 时间范围 | PST 文件名示例 |
|------|----------|----------------|
| Q1 | 1月 - 3月 | `2024_Q1.pst` |
| Q2 | 4月 - 6月 | `2024_Q2.pst` |
| Q3 | 7月 - 9月 | `2024_Q3.pst` |
| Q4 | 10月 - 12月 | `2024_Q4.pst` |

> 💡 **示例**：一封发送于 2024 年 5 月 15 日的邮件，无论何时执行归档，都会被归档到 `2024_Q2.pst` 中。

**文件夹过滤**：

程序支持两种文件夹过滤模式，通过 `archive_mode` 配置项控制：

- **`all` 模式**（默认）：归档所有文件夹，但排除 `exclude_folders` 列表中指定的文件夹。
- **`list` 模式**：仅归档 `include_folders` 列表中指定的文件夹。

配置示例：

```yaml
# all 模式：归档所有文件夹，但排除"重要邮件"和"项目资料"
archive_mode: all
exclude_folders:
  - "重要邮件"
  - "项目资料"

# list 模式：仅归档"收件箱"和"已发送邮件"
archive_mode: list
include_folders:
  - "收件箱"
  - "已发送邮件"
```

> 💡 **提示**：文件夹名称支持使用 `/` 分隔的子文件夹路径，如 `"项目/2024/Q1"`。

### 2.2 全量整理

全量整理是一个深度操作，适用于首次部署或需要修复历史归档问题的场景。

#### 何时使用

- 🆕 **首次部署**：需要一次性归档邮箱中所有历史邮件
- 🔀 **第三方迁移**：需要将旧的第三方 PST 文件迁移到新的归档体系
- 🔧 **分类修复**：已有的 PST 中存在邮件被归档到错误季度的情况

#### 两阶段执行说明

全量整理分为两个阶段自动执行：

**阶段一：全量归档**
- 暂停日常定时任务
- 忽略 `retain_days` 设置，扫描并归档邮箱中**所有**超过安全延迟的邮件
- 每个轮询周期处理 `max_batch_size` 封邮件，自动循环直到全部处理完成

**阶段二：PST 纠偏 + 第三方迁移**
- 遍历所有已挂载的 PST 文件（以及 `legacy_pst_scan_paths` 中指定的路径）
- 检查每封邮件的实际时间与当前所在 PST 的季度是否匹配
- 将错位的邮件迁移到正确的季度 PST 中
- 自动清理迁移后变空的 PST 文件

#### 如何触发

通过以下任一方式触发全量整理：

- **系统托盘**：右键菜单选择 🧹 **全量整理**
- **Web 控制台**：在仪表盘页面点击 **全量整理** 按钮

> ⚠️ **注意**：全量整理会处理大量邮件，耗时较长。执行期间部分操作会被禁用。请耐心等待完成。

#### 第三方 PST 迁移

如果您之前使用过其他归档工具生成了 PST 文件，可以通过 `legacy_pst_scan_paths` 配置项指定这些 PST 文件的路径：

```yaml
legacy_pst_scan_paths:
  - "D:\\旧归档"
  - "E:\\Backup\\PST"
```

> 💡 **提示**：如果 `include_mounted_psts` 设置为 `true`（默认），程序还会自动扫描 Outlook 中已挂载的 PST 文件，无需手动指定路径。

### 2.3 邮件还原

邮件还原功能可以将已归档到 PST 中的邮件迁移回 Exchange 邮箱。

#### 操作步骤

1. 通过以下任一方式打开还原功能：
   - **系统托盘**：右键菜单选择对应选项
   - **Web 控制台**：在仪表盘页面点击 **还原数据** 按钮

2. 在弹出的确认窗口中，选择还原选项：
   - ☑️ **删除空 PST**：还原后自动清理变空的 PST 文件
   - ☑️ **删除重复邮件**：自动跳过 Exchange 邮箱中已存在的邮件

3. 确认后开始执行还原操作。

#### 去重机制

还原时程序会自动检测重复邮件，避免同一封邮件在 Exchange 和 PST 中各存在一份。当选择"删除重复邮件"选项时，已存在于 Exchange 中的邮件会被自动跳过。

#### 注意事项

> ⚠️ **OST 容量限制**：Exchange 邮箱的本地缓存（OST 文件）有 **50GB** 的容量上限。如果归档了大量邮件，请确保 Exchange 邮箱有足够空间接收还原的邮件，否则可能导致 OST 文件损坏。

> 💡 **建议**：还原前先在 Web 控制台查看 PST 挂载列表，了解各 PST 文件的大小，评估总还原量是否在 OST 容量范围内。

### 2.4 系统托盘

程序运行后会在系统托盘区域显示图标，提供便捷的右键菜单操作。

#### 菜单项说明

| 图标 | 菜单项 | 说明 |
|------|--------|------|
| 🚀 | **立即执行一次** | 手动触发一次归档任务（受 `max_batch_size` 限制） |
| 🧹 | **全量整理** | 暂停日常任务，执行深度扫描与纠偏（详见 [2.2 全量整理](#22-全量整理)） |
| ─── | ─── | 分隔线 |
| 📂 | **打开日志目录** | 用文件资源管理器打开 `logs/` 文件夹 |
| 📝 | **打开配置文件** | 用系统默认编辑器打开 `config.yaml` |
| 🔄 | **重新加载配置** | 热重载配置文件，无需重启程序 |
| ☑ | **Web 配置中心** | 启动/停止嵌入式 Web 服务 |
| ☑ | **开机自启** | 切换开机自启动状态 |
| ─── | ─── | 分隔线 |
| ❌ | **退出** | 优雅关闭程序，释放 COM 资源 |

#### 状态指示

程序每 30 秒自动检测并更新托盘状态：

| 状态 | 图标变化 | 说明 |
|------|----------|------|
| 🟢 运行中 | 动态图标 | 正在执行归档任务 |
| ⚪ 空闲 | 默认图标 | 等待下一次定时扫描 |
| 🟡 磁盘警告 | 带警告标记 | 磁盘空间不足，详见 [磁盘监控](#磁盘监控) |
| 🔒 操作中 | 部分菜单禁用 | 全量整理进行中，部分操作暂时不可用 |

### 2.5 Web 控制台

Web 控制台是一个嵌入式的 Web 管理界面，提供图形化的配置和操作能力。

#### 如何打开

1. 右键系统托盘图标，选择 ☑ **Web 配置中心** 启动 Web 服务。
2. 在浏览器中访问：**https://127.0.0.1:8443**

> 💡 **提示**：Web 服务使用自签名证书，浏览器可能会提示"不安全"，请点击"高级"→"继续访问"即可。

#### 仪表盘说明

仪表盘页面包含以下区域：

**操作卡片**：
- 🚀 **立即执行** — 触发一次归档
- 🧹 **全量整理** — 执行深度扫描与纠偏
- 🔄 **重载配置** — 热重载配置文件
- 📂 **日志目录** — 打开日志文件夹
- 📝 **配置文件** — 打开 config.yaml
- ↩️ **还原数据** — 将 PST 邮件还原到 Exchange
- ❌ **退出程序** — 关闭程序

**PST 挂载列表**：
- 实时显示 Outlook 中当前已挂载的所有 PST 文件
- 包含文件名、路径和文件大小等信息

**实时日志终端**：
- 通过 SSE（Server-Sent Events）连接实时显示运行日志
- 不同级别的日志使用不同颜色标识：
  - 🟢 INFO — 正常信息
  - 🟡 WARN — 警告信息
  - 🔴 ERROR — 错误信息
  - ⚪ DEBUG — 调试信息（需开启 `debug_log`）

#### 配置中心说明

配置中心页面允许您在线查看和修改所有配置项，修改后自动保存并生效。

**基本配置**：
- PST 根目录
- 轮询间隔
- 安全延迟
- 保留天数
- 单次最大处理数

**高级配置**：
- 邮件移动间隔
- 日志保留天数
- 历史 PST 扫描目录
- Dry Run / Copy Only / Debug 开关

**文件夹过滤**：
- 归档模式切换（全量/白名单）
- 排除/包含文件夹列表

> 💡 **提示**：配置中心的修改会自动保存，无需手动点击保存按钮。

#### 还原弹窗

在 Web 控制台点击"还原数据"后会弹出确认窗口：

- ⚠️ 显示 OST 容量 50GB 上限的警告信息
- 提供选项：
  - ☑️ 删除空 PST — 还原后自动清理空的 PST 文件
  - ☑️ 删除重复邮件 — 跳过已存在于 Exchange 的邮件

#### 心跳机制

- 前端每 **30 秒**发送一次心跳请求
- 如果后端 **5 分钟**未收到心跳，会自动关闭 Web 服务以节省资源
- 重新打开 Web 控制台时会自动重启 Web 服务

### 2.6 开机自启

程序支持配置 Windows 开机自动启动，无需每次手动运行。

#### 如何设置

**方式一：系统托盘**

右键系统托盘图标，点击 ☑ **开机自启**，菜单项前出现 ✅ 标记表示已开启。

**方式二：Web 控制台**

在配置中心页面找到开机自启开关，点击切换。

> 💡 **原理说明**：程序通过修改 Windows 注册表（`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`）实现开机自启，不创建额外的服务进程。设置后，每次 Windows 登录时会自动启动程序。

#### 如何取消

再次点击系统托盘菜单或 Web 控制台中的 ☑ **开机自启** 即可关闭。

### 2.7 配置文件详解

配置文件 `config.yaml` 位于程序同目录下，首次运行时自动生成。以下是每个配置项的详细说明。

#### 基础配置

```yaml
# PST 文件存储根目录
# 所有归档生成的 PST 文件都将存放在此目录下
pst_root_path: "./OutlookArchives"
```

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `pst_root_path` | string | `./OutlookArchives` | PST 文件存储根目录。可以使用相对路径（相对于程序目录）或绝对路径。建议选择空间充足的磁盘分区。 |

#### 归档调度配置

```yaml
# 自动扫描轮询间隔（分钟）
# 程序每隔多久检查一次邮箱并执行归档
poll_interval_minutes: 10

# 安全延迟时间（分钟）
# 邮件需要等待多久后才会被归档，防止与 Exchange 同步冲突
safe_delay_minutes: 10

# 保留近期邮件天数
# 只归档超过此天数的邮件，0 表示不保留（归档所有邮件）
retain_days: 30

# 单次最大处理邮件数量
# 每次归档扫描最多处理多少封邮件，避免单次操作耗时过长
max_batch_size: 500
```

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `poll_interval_minutes` | int | `10` | 自动扫描间隔。值越小扫描越频繁，但会增加系统资源消耗。建议 5-30 分钟。 |
| `safe_delay_minutes` | int | `10` | 安全延迟。防止刚收到的邮件被立即归档，给 Exchange 同步留出缓冲时间。设为 0 可禁用。 |
| `retain_days` | int | `30` | 保留天数。只归档超过此天数的邮件。设为 0 表示归档所有邮件（慎用）。 |
| `max_batch_size` | int | `500` | 单次批量大小。避免一次处理过多邮件导致 Outlook 响应缓慢。设为 0 表示不限制。 |

#### 高级配置

```yaml
# 移动单封邮件后的休眠时间（毫秒）
# 降低 Outlook 响应压力，避免界面卡顿
move_interval_ms: 50

# 日志文件保留天数
# 超过此天数的旧日志文件会被自动清理

# 流式遍历邮件时的分块大小
# 控制每次从 Outlook 加载多少封邮件的元数据
stream_block_size: 1000
```

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `move_interval_ms` | int | `50` | 移动间隔（毫秒）。每移动一封邮件后暂停一小段时间，减少 Outlook 卡顿。 |
| `log_retention_days` | int | `7` | 日志保留天数。超过此天数的日志文件会被自动删除。 |
| `stream_block_size` | int | `1000` | 分块遍历大小。控制批量读取邮件元数据的数量，一般无需修改。 |

| `include_folders` | []string | `[]` | 包含列表。在 `list` 模式下，仅归档列表中的文件夹。 |

## 3. 使用场景与最佳实践

### 3.1 场景一：邮箱即将满仓

**症状**：收到 Exchange 邮箱空间不足的警告，无法接收新邮件。

**解决方案**：

1. 先以 **Dry Run** 模式运行，确认归档计划：

   ```yaml
   dry_run: true
   retain_days: 7    # 临时降低保留天数，快速释放空间
   max_batch_size: 1000  # 增大批次，加快处理
   ```

2. 确认日志输出正确后，关闭 Dry Run 正式执行：

   ```yaml
   dry_run: false
   ```

3. 通过托盘菜单点击 🚀 **立即执行一次**，快速处理一批邮件。

4. 空间释放后，恢复常规配置：

   ```yaml
   retain_days: 30
   max_batch_size: 500
   ```

> 💡 **应急提示**：如果邮箱已经完全无法收发邮件，可以将 `retain_days` 设为 `0`，归档所有邮件。但建议先用 Dry Run 验证。

### 3.2 场景二：定期归档维护

**需求**：希望程序在后台安静运行，定期自动归档历史邮件。

**推荐配置**：

```yaml
# 每 15 分钟检查一次
poll_interval_minutes: 15

# 保留近 60 天的邮件
retain_days: 60

# 单次处理 500 封
max_batch_size: 500

# 安全延迟 10 分钟
safe_delay_minutes: 10
```

**最佳实践**：
- ✅ 开启 **开机自启**，确保程序随系统启动
- ✅ 定期检查日志，确认归档正常运行
- ✅ 监控 PST 存储目录的磁盘空间

### 3.3 场景三：迁移历史 PST

**需求**：之前使用其他工具生成了多个 PST 文件，希望统一管理。

**操作步骤**：

1. 配置第三方 PST 扫描路径：

   ```yaml
   legacy_pst_scan_paths:
     - "D:\\旧归档"
     - "E:\\Backup\\OldPST"
   include_mounted_psts: true
   ```

2. 触发全量整理：右键托盘菜单 → 🧹 **全量整理**

3. 程序会自动：
   - 扫描指定目录下的所有 PST 文件
   - 将 Outlook 已挂载的 PST 纳入整理范围
   - 按邮件实际时间重新分类到正确的季度 PST
   - 清理迁移后变空的旧 PST 文件

### 3.4 场景四：需要还原邮件

**需求**：发现某些邮件被误归档，需要还原到 Exchange 邮箱。

**操作步骤**：

1. 先在 Outlook 中确认需要还原的邮件所在的 PST 文件和文件夹。
2. 通过托盘菜单或 Web 控制台打开还原功能。
3. 勾选 **删除重复邮件** 选项（推荐）。
4. 确认执行还原。

> ⚠️ **注意**：还原前请确认 Exchange 邮箱（OST）有足够的剩余空间。OST 上限为 50GB。

### 3.5 安全建议

- 🔒 **首次使用必做 Dry Run**：永远先用模拟模式验证，再正式执行。
- 💾 **定期备份 PST 文件**：PST 文件是您的邮件备份，请定期将 `OutlookArchives` 目录备份到外部存储。
- 📊 **监控磁盘空间**：确保 PST 存储目录所在磁盘有足够空间（建议至少 5GB 余量）。
- 🔄 **使用 Copy Only 模式迁移**：从其他系统迁移时，可先用 `copy_only: true` 模式测试，确认无误后再改为移动模式。
- 📝 **保留日志**：适当增加 `log_retention_days`，方便事后追溯操作记录。

## 4. 常见问题 (FAQ)

### Q: 程序无法启动怎么办？

**A**: 请按以下顺序排查：

1. ✅ 确认 Classic Outlook 已启动并正常运行（不是"新版 Outlook"）
2. ✅ 检查是否有另一个程序实例已在运行（程序有防多开保护）
3. ✅ 查看 `logs/archiver.log` 中是否有错误信息
4. ✅ 确认程序有足够的文件读写权限

详见 [5. 故障排除](#5-故障排除)。

### Q: 归档后 Outlook 侧边栏看不到 PST？

**A**: 归档生成的 PST 文件默认不会自动在 Outlook 中显示。您需要手动打开：

1. 打开 Outlook
2. 点击 **文件** → **打开和导出** → **打开 Outlook 数据文件**
3. 浏览到 PST 存储目录（默认为程序目录下的 `OutlookArchives`）
4. 选择对应的 PST 文件（如 `2024_Q2.pst`）
5. PST 文件会出现在 Outlook 侧边栏中

### Q: 邮件被归档到错误的季度？

**A**: 本程序严格根据邮件的**实际发送/接收时间**（而非系统时间）来计算季度。如果您认为分类有误：

1. 开启 `debug_log: true` 查看详细日志，确认邮件的原始时间戳
2. 使用 🧹 **全量整理** 功能，程序会自动检测并纠偏已有 PST 中的错误分类
3. 如果是第三方工具造成的分类错误，配置 `legacy_pst_scan_paths` 后执行全量整理

### Q: 磁盘空间不足会怎样？

**A**: 程序内置了磁盘空间监控机制：

| 剩余空间 | 告警级别 | 影响 |
|----------|----------|------|
| < 1 GB | ⚠️ 警告 | 托盘图标显示警告标记，日志输出警告 |
| < 500 MB | 🔴 严重 | **拒绝执行**任何归档操作，托盘图标显示严重警告 |

> 💡 **建议**：定期检查 PST 存储目录所在磁盘的剩余空间，建议保持至少 5GB 余量。

### Q: 如何暂停归档？

**A**: 有以下几种方式：

1. **临时暂停**：编辑 `config.yaml`，将 `poll_interval_minutes` 设置为一个很大的值（如 `99999`），然后重新加载配置。
2. **模拟模式**：将 `dry_run` 设置为 `true`，程序会继续扫描但不实际移动邮件。
3. **退出程序**：右键托盘菜单 → ❌ **退出**。

> 💡 **提示**：没有"暂停"按钮是因为设计上程序应该是无人值守的后台服务。如果只是临时需要，建议用 Dry Run 模式。

### Q: Dry Run 模式下日志在哪里看？

**A**: 日志文件位于程序目录下的 `logs/archiver.log`。您也可以：

- 通过托盘菜单点击 📂 **打开日志目录**
- 通过 Web 控制台的**实时日志终端**查看

Dry Run 模式的日志行会带有 `[DryRun]` 标记，例如：
```
[INFO] [DryRun] Would move: "会议纪要" (2024-03-15) → OutlookArchives/2024_Q1.pst
```

### Q: 全量整理需要多长时间？

**A**: 取决于以下因素：

| 因素 | 影响 |
|------|------|
| 邮箱中的邮件总量 | 邮件越多，耗时越长 |
| PST 文件数量和大小 | 第三方 PST 越多，纠偏耗时越长 |
| `max_batch_size` 设置 | 批次越大，单轮处理越快但 Outlook 更卡 |
| `move_interval_ms` 设置 | 间隔越短处理越快，但 Outlook 更卡 |

**参考数据**：处理 10,000 封邮件大约需要 30-60 分钟（默认配置下）。

> 💡 **建议**：全量整理期间请避免在 Outlook 中进行大量操作，以减少 COM 冲突。

### Q: 还原后邮件重复了怎么办？

**A**: 如果还原时没有勾选"删除重复邮件"选项，可能会出现重复。解决方法：

1. 重新执行还原操作，这次勾选 ☑️ **删除重复邮件** 选项
2. 程序会自动检测并跳过已存在于 Exchange 中的邮件

### Q: 如何备份 PST 文件？

**A**: PST 文件就是普通的 Outlook 数据文件，备份方法：

1. **直接复制**：将整个 `OutlookArchives` 目录复制到外部存储设备
2. **注意**：备份前建议先关闭 Outlook，或者确保 Outlook 没有正在写入该 PST 文件

> ⚠️ **重要**：PST 文件是您邮件的唯一本地备份，请务必定期备份！

### Q: 支持新版 Outlook 吗？

**A**: **不支持**。本程序仅支持基于 COM 接口的 **Classic Outlook**（经典版 Outlook），包括：

- ✅ Microsoft Office LTSC 2024 专业增强版
- ✅ 经典版 Classic Outlook

不支持：
- ❌ 基于 WebView2 的"新版 Outlook"（New Outlook）

> 💡 如何判断：Classic Outlook 的标题栏显示"Outlook"，启动时有传统的加载界面；新版 Outlook 的标题栏显示"Outlook (new)"，界面更现代化。

### Q: 日志文件在哪里？

**A**: 日志文件位于程序目录下的 `logs/` 文件夹中：

```
📁 程序目录/
└── logs/
    ├── archiver.log       # 当前运行日志
    ├── archiver.log.2024-06-01  # 自动轮转的历史日志
    └── archiver.log.2024-06-02
```

- 当前日志：`logs/archiver.log`
- 历史日志：按日期自动轮转，保留天数由 `log_retention_days` 控制
- 快速打开：托盘菜单 → 📂 **打开日志目录**

## 5. 故障排除

### Outlook 未运行

**症状**：程序启动后日志显示无法连接到 Outlook。

**解决步骤**：

1. 确认 Classic Outlook 已启动且处于正常运行状态
2. 确认 Outlook 已登录 Exchange 邮箱
3. 如果 Outlook 弹出了登录或密码输入框，请先完成登录
4. 重启程序

### COM 错误

**症状**：日志中出现 `COM error`、`OLE error` 或类似错误信息。

**可能原因**：
- Outlook 正在执行其他操作（如收发邮件、同步文件夹）
- Outlook 进程不稳定

**解决步骤**：

1. 等待几分钟后重试
2. 如果频繁出现，尝试重启 Outlook
3. 开启 `debug_log: true` 获取更详细的错误信息
4. 检查 `move_interval_ms` 是否设置过小（建议不小于 50）

### PST 文件损坏

**症状**：Outlook 无法打开 PST 文件，或打开后显示错误。

**预防措施**：
- 不要在归档过程中强制关闭程序或 Outlook
- 确保磁盘空间充足，避免写入过程中空间不足
- 定期备份 PST 文件

**修复步骤**：

1. 使用 Outlook 自带的修复工具：
   - 找到 Outlook 安装目录下的 `SCANPST.EXE`
   - 运行并选择损坏的 PST 文件进行修复
2. 如果修复失败，从备份中恢复 PST 文件

### 端口被占用

**症状**：Web 控制台无法启动，日志显示端口 8443 被占用。

**解决步骤**：

1. 查找占用端口的进程：

   ```powershell
   Get-NetTCPConnection -LocalPort 8443 | Select-Object OwningProcess | ForEach-Object { Get-Process -Id $_.OwningProcess }
   ```

2. 如果是其他程序占用，关闭该程序或修改其端口配置
3. 如果是本程序的残留进程，结束该进程后重启程序

> 💡 **提示**：Web 服务使用端口 **8443**，如果该端口被持续占用，可能需要重启电脑。

## 6. 术语表

| 术语 | 全称 | 说明 |
|------|------|------|
| **PST** | Personal Storage Table | Outlook 个人文件夹文件，用于本地存储邮件、日历、联系人等数据。本程序将邮件归档到 PST 文件中。 |
| **OST** | Offline Storage Table | Outlook 离线数据文件，是 Exchange 邮箱在本地的缓存副本。有 50GB 容量上限。 |
| **MAPI** | Messaging Application Programming Interface | 微软的消息应用程序编程接口，是 Outlook 底层通信的协议基础。 |
| **COM** | Component Object Model | 微软的组件对象模型技术，本程序通过 COM 接口与 Outlook 进行自动化交互。 |
| **SSE** | Server-Sent Events | 服务器推送事件技术，Web 控制台通过 SSE 实时推送日志到浏览器端。 |
| **Exchange** | Microsoft Exchange Server | 微软的企业邮件服务器，管理用户的邮箱数据。Outlook 作为 Exchange 的客户端。 |
| **Dry Run** | 模拟运行 | 程序执行完整的分析和日志记录，但不实际移动邮件。用于验证归档计划是否正确。 |
| **Copy Only** | 仅复制模式 | 邮件只复制到 PST 文件中，不从 Exchange 邮箱删除。相当于"备份"而非"迁移"。 |
| **全量整理** | — | 一次性归档所有邮件 + 纠偏已有 PST 中的错误分类 + 迁移第三方 PST 的综合操作。 |
| **热重载** | Hot Reload | 修改配置文件后无需重启程序，通过菜单操作即可使新配置生效。 |
| **CORS** | Cross-Origin Resource Sharing | 跨域资源共享，Web 安全机制。本程序的 Web 服务仅监听本地地址。 |
| **LTSC** | Long-Term Servicing Channel | 微软 Office 的长期服务频道版本，提供稳定的经典功能集。 |
